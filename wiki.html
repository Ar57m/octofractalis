Compilation

This fractal generator is cross-platform and should work on most systems. It has been tested on the following environments:

  • Android (via Termux)
  • Linux (Debian, Ubuntu)
  • Windows 11 (with w64devkit)

Installation Requirements

Before building the generator, ensure you have the required Python packages:

pip install numpy opencv-python

Note for Termux users: Additional steps are needed to support OpenCV:

apt install x11-repo && apt update
  apt install opencv
  apt install opencv-python

GPU and Performance Flags

The generator supports optional flags to enhance performance:

  • --cuda: Enables GPU acceleration on NVIDIA GPUs for faster computation.
  • --f32: Uses float32 precision in most operations, which may improve speed on some systems.
  • --octo: Activates Octonion math for fractal generation.

Build Instructions

Clone the repository and run the build script:

git clone https://github.com/Ar57m/another_fractal_generator
  cd another_fractal_generator
  bash build.sh

Note: The build.sh script compiles the generator. Once compiled, you can run the Python script directly to start generating fractals.

Fractal Types

Mandelbrot
Uses the classic iteration z = z² + c to determine escape.
Variables used:
• z: the dynamic point in the fractal space.
• c: calculated from the pixel coordinates (x, y) and used as a constant.
• it: the iteration count driving the loop.
• x and y: pixel coordinates.
• Also constants like phi, pi, and e can be used.

Parameters used:
• Gradient, Fast Mode, Fractalize Image.
• General Parameters, Initial Z Parameters, Palette settings and Coordinates.
Note: Escape radius when set to zero it's actually using 2.

Julia Set
Similar to Mandelbrot but with a fixed c provided externally.
Variables used:
• z: updated at each iteration.
• c: fixed constant defining the set.
• it: iteration counter.
• x and y: pixel coordinates.
• Also constants like phi, pi, and e can be used.

Parameters used:
• Gradient, Fast Mode, Fractalize Image.
• General Parameters, Initial Z Parameters, Julia Set Parameters, Palette settings and Coordinates.
Note: Escape radius when set to zero it's actually using 2.

Newton
Applies Newton’s method to find roots of complex functions.
Variables used:
• z: the current estimate of the root.
• c: used when in Julia mode (Newton JuliaSet variant).
• it: counts iterations for convergence.
• x and y: serve as pixel coordinates.
• Also constants like phi, pi, and e can be used.

Parameters used:
• Gradient, Newton Epsilon, Fractalize Image.
• General Parameters, Initial Z Parameters, Palette settings and Coordinates.

Newton Julia Set
A hybrid of Newton’s method and the Julia set, where a fixed c is used.
Variables used:
• z: iteratively updated using Newton’s method.
• c: fixed as in Julia mode.
• it, x, and y: track iterations and pixel positions.
• Also constants like phi, pi, and e can be used.

Parameters used:
• Gradient, Newton Epsilon, Fractalize Image.
• General Parameters, Initial Z Parameters, Julia Set Parameters, Palette settings and Coordinates.

Magnet Pendulum
Simulates a chaotic pendulum influenced by multiple magnets.
Variables used:
• z: current position in space.
• v: the velocity of the pendulum.
• f: the force acting on the pendulum, computed from attractor influences.
• dif: the difference between z and an attractor.
• Also constants like phi, pi, and e can be used.

Parameters used:
• Gradient, Number of Magnets, Fast Mode, Fractalize Image.
• General Parameters, Initial Z Parameters, Palette settings and Coordinates.
Note: Start with the equation z+v and gradient 0; Escape radius when set to zero it's actually using 9.

Lorenz Attractor
Generates trajectories based on the Lorenz system’s differential equations.
Variables used:
• z: represents the current point in the trajectory.
• it: the iteration count.
• dx, dy, dz: control the spatial resolution and scaling.
• dt: the time step for integration.
• Also constants like phi, pi, and e can be used.

Parameters used:
• Gradient, Fractalize Image.
• General Parameters, Initial Z Parameters, Palette settings, Lorenz Parameters and Coordinates.
Note: Start with the equation dx+dy*i+dz*j, coordinates at [-30, 30, -30, 30], z initial[0.05 ,1.0] and Max Iterations are used as the number of points.

Lyapunov
Computes fractals based on Lyapunov exponents to illustrate regions of stability and chaos.
Variables used:
• v: a variable representing an iterative value that alternates in function.
• p: an accumulator that sums expression results over iterations.
• c: used to switch between different operations during iterations.
• it, x, and y: track iteration and pixel positions.
• Also constants like phi, pi, and e can be used.

Parameters used:
• Gradient, Fractalize Image.
• General Parameters, Initial Z Parameters, Palette settings and Coordinates.
Note: Try starting with the equation log(mag(c*(1-2*v))), coordinates at [-8, 8, -8, 8]; Escape radius when set to zero it's actually using 400.

Sandpile
Models self-organized criticality where grains topple once a threshold is exceeded.
Note: This fractal type does not use the custom equation parser; it directly simulates grain distribution and toppling.

Parameters used:
• Gradient, Sandpile Max Grains, Fractalize Image.
• General Parameters(except Escape Radius and Prime Array Size) and Palette settings(except lake).

Operators/Operands


+ (Unary)
Returns the factor unchanged. For example, writing +x simply yields x.

- (Unary)
Negates the following factor, returning its additive inverse. For example, -x produces the negative of x.

+ (Binary)
Adds two operands. For example, a + b computes the sum of a and b.

- (Binary)
Subtracts the right operand from the left. For example, a - b calculates the difference.

*
Multiplies two operands. For example, a * b computes their product.

/
Divides the left operand by the right operand. For example, a / b yields the quotient.

%
Calculates the remainder when the left operand is divided by the right operand. For instance, a % b returns the fmod for each component.
For example:
(4+6i+8j+10k)%(3+3i+3j+3k) returns (1+0i+2j+1k)

^
Raises the left operand to the power of the right operand. For example, a ^ b computes a raised to b.

=
Tests **approximate equality** between two operands by comparing their components with a small tolerance (MSE < 1e-9). Returns 1 if they are nearly equal, otherwise 0. The result is returned as a quaternion/octonion with only the real part set.
Example: a = b returns 1 (as a quaternion) if a and b are almost equal, otherwise 0.

&
Computes the cosine similarity between two operands, returning a real value. For example, a & b calculates the cosine similarity.

>
Compares the **magnitudes** of two operands. Returns 1 (as a quaternion/octonion) if the left operand’s magnitude is greater than the right’s, otherwise 0.
Example: a > b checks if the magnitude of a is greater than b.

<
Compares the **magnitudes** of two operands. Returns 1 (as a quaternion/octonion) if the left operand’s magnitude is less than the right’s, otherwise 0.
Example: a < b checks if the magnitude of a is less than b.



Functions

The fractal generator’s parser supports a wide range of functions for quaternion/octonion math. These functions are called using the syntax functionName(arg1, arg2, arg3). Below is a list of each supported function along with a short description and an example of how to use it.


abs(x)
Returns a new quaternion/octonion whose components are the absolute values of those in x.

airy(x)
Computes the Airy function of x using a series expansion.

arg(x)
Returns the argument (phase) of x, computed from its real and imaginary magnitude using atan2.

circle(x, radius)
Uses the angle from x to generate a point on a circle of the specified radius. (Result: (cos(angle)*radius, sin(angle)*radius)).

conj(x)
Returns the conjugate of x by negating its imaginary components.

cos(x)
Computes the cosine of x using quaternion/octonion cosine formulas.

cosh(x)
Computes the hyperbolic cosine of x.

diff(f(x))
Computes the numerical derivative of the expression f(x) using a finite difference approximation.

exp(x)
Returns the exponential (e^x) of x.

i(x)
Extracts the first imaginary component (often labeled as imag) of x.

if(cond, a, b)
A ternary function that returns a if the condition cond is met (typically by checking if the absolute of the real part is bigger than 1e-9) and b otherwise.

im(x)
Returns a quaternion/octonion composed of the imaginary components of x (excluding its real part).

j(x)
Extracts the j component from x.

k(x)
Extracts the k component from x.

l(x)
(Octonion only) Extracts the l component from x.

log(x)
Returns the natural logarithm (base e) of x.

logn(x, base)
Computes the logarithm of x with the specified base.

m(x)
(Octonion only) Extracts the m component from x.

mag(x)
Returns the magnitude (norm) of x.

max(x, y)
Returns the value with the larger magnitude between x and y.

min(x, y)
Returns the value with the smaller magnitude between x and y.

n(x)
(Octonion only) Extracts the n component from x.

o(x)
(Octonion only) Extracts the o component from x.

pow(x, y)
Raises x to the power of y. The same as doing (x^y).

rand(a, b, seed)
(CPU only) Generates a random quaternion/octonion within the range defined by a and b, using seed for initialization.

re(x)
Returns the real component of x as a quaternion/octonion.

root(x, n)
Computes the nth root of x, equivalent to pow(x, 1/n).

rotate(x, angle, axis)
Rotates x by a given angle around the specified axis(an integer) (using circular rotation).
For quaternions these are the axis:
0: (real, imag)
1: (real, j)
2: (real, k)
3: (imag, j)
4: (imag, k)
5: (j, k)

For octonions:
0: (real, imag)
1: (real, j)
2: (real, k)
3: (real, l)
4: (real, m)
5: (real, n)
6: (real, o)
7: (imag, j)
8: (imag, k)
9: (imag, l)
10: (imag, m)
11: (imag, n)
12: (imag, o)
13: (j, k)
14: (j, l)
15: (j, m)
16: (j, n)
17: (j, o)
18: (k, l)
19: (k, m)
20: (k, n)
21: (k, o)
22: (l, m)
23: (l, n)
24: (l, o)
25: (m, n)
26: (m, o)
27: (n, o)

rotation(x, angle, axis)
Performs a rotation of x using an alternative rotation algorithm. Parameters are similar to rotate.

round(x)
Rounds each component of x to the nearest integer.

sign(x)
Returns the sign of each component of x while preserving its quaternion/octonion structure.

sin(x)
Computes the sine of x.

sinh(x)
Computes the hyperbolic sine of x.

sqrt(x)
Returns the square root of x.

square(x, sideLength)
Generates a square projection based on the angle of x and the specified sideLength.

tan(x)
Computes the tangent of x.

tanh(x)
Computes the hyperbolic tangent of x.

triangle(x, sideLength)
Creates a triangle projection based on x with the specified sideLength.

zeta(x)
Computes the Riemann zeta function of x using a series expansion.


For example, to raise a quaternion to a power, you would write: pow(base, exponent) or (base^exponent).
